# Codebase Indexing Service

Этот сервис предназначен для локального индексирования кодовой базы различных проектов c `Kilo Code`, `Qdrant`, `Ollama`.

Конфигурация изолирована и может быть использована как отдельный инструмент.

## Состав сервисов
- **Qdrant**: Векторная база данных для хранения эмбеддингов кода.
  - **HTTP Port**: 10500
  - **gRPC Port**: 10501
- **Ollama**: Локальный запуск LLM для генерации эмбеддингов или анализа.
  - **Port**: 11434

## Быстрый старт
Для запуска сервисов перейдите в эту директорию и выполните:
```bash
docker-compose up -d
```
После запуска необходимо скачать модель для эмбеддингов:
```bash
docker exec ollama-dev ollama pull nomic-embed-text
```

## Настройка в Kilo.ai
Для активации индексации в GUI Kilo.ai используйте следующие параметры:

### Провайдер эмбеддера (Ollama)
- **Базовый URL Ollama**: `http://localhost:11434`
- **Модель**: `nomic-embed-text`
- **Размерность модели**: `768`

### Поставщик векторного хранилища (Qdrant)
- **URL Qdrant**: `http://localhost:10500`
- **Qdrant Dashborad**: `http://localhost:10500/dashboard`
- **API-ключ Qdrant**: (необязательно, оставить пустым)

## Хранение данных
Данные сохраняются в именованных Docker volumes:
- `qdrant_indexing_storage` — векторы и метаданные кода.
- `ollama_data` — скачанные модели Ollama.

Это позволяет сохранять индекс даже после удаления контейнеров.
